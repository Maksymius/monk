# Blueprint: Лендинг 'Научись Быть'

## 1. Обзор проекта

**Цель:** Создание современного, визуально привлекательного и производительного лендинга для курса по развитию осознанности.

**Технологический стек:**
*   **Frontend:** HTML5, CSS3, JavaScript (ES-модули)
*   **Backend (для форм):** Firebase (Firestore)
*   **Принципы:** Mobile-first, Progressive Enhancement, Accessibility (A11Y).

## 2. Структура проекта

Проект имеет следующую структуру для четкого разделения ресурсов:

```
/
|-- src/
|   |-- assets/
|   |   |-- img/ (фоновые изображения)
|   |-- css/
|   |   |-- style.css (основные стили)
|   |-- js/
|   |   |-- app.js (главный файл, точка входа)
|   |   |-- firebase-config.js (конфигурация Firebase)
|   |   |-- ui/ (модули, отвечающие за UI)
|   |   |   |-- cursor.js
|   |   |   |-- loader.js
|   |   |   |-- parallax.js
|   |   |   |-- reveal.js
|   |   |-- firebase/ (модули для работы с Firebase)
|   |   |   |-- forms.js
|   |-- index.html (главная страница)
|-- firebase.json (конфигурация хостинга Firebase)
|-- .firebaserc (конфигурация проекта Firebase)
|-- blueprint.md (этот документ)
```

## 3. Реализованный функционал и дизайн

### 3.1. Визуальный дизайн и UX

*   **Типографика:** Использованы шрифты `Montserrat` для заголовков и `Roboto` для основного текста для создания контраста и улучшения читаемости.
*   **Цветовая палитра:** Применяются CSS-переменные для легкой смены темы. Палитра включает теплые оранжевые акценты, глубокие темные и чистые светлые тона.
*   **Эффект "Glow" курсора:** Интерактивный светящийся элемент, следующий за курсором мыши, для создания "вау-эффекта" на десктопных устройствах (отключается на мобильных).
*   **Прелоадер:** Стильный и плавный SVG-спиннер, который скрывается после полной загрузки страницы.
*   **Parallax-эффект:** Фоновые изображения в секциях имеют эффект параллакса при прокрутке для создания глубины (отключается на мобильных для производительности).
*   **Анимация появления:** Элементы плавно появляются при скролле страницы, используя `IntersectionObserver` API для высокой производительности.
*   **Микровзаимодействия:** Кнопки и формы имеют ховер-эффекты (тени, трансформации) для улучшения тактильной обратной связи.
*   **Текстура:** На темные секции добавлена легкая "шумовая" текстура для придания премиального вида.

### 3.2. Техническая реализация

*   **Модульный JavaScript:** Весь JS-код разбит на ES-модули по принципу единой ответственности. Главный файл `app.js` координирует инициализацию всех UI-компонентов и обработчиков форм.
*   **Оптимизация производительности:**
    *   Использование `requestAnimationFrame` для анимации курсора.
    *   Отключение ресурсоемких анимаций (параллакс, курсор) на мобильных устройствах.
    *   Отложенная загрузка изображений (неявная, через `data-bg`).
    *   Использование `IntersectionObserver` вместо постоянного прослушивания события скролла.
    *   Импорт Firebase SDK через ES-модули для загрузки только необходимых функций.
*   **Интеграция с Firebase:**
    *   Обе формы ("Получить программу" и "Проснуться") асинхронно отправляют данные в **Firebase Firestore**.
    *   Созданы отдельные коллекции (`offer-requests` и `contact-requests`) для хранения данных.
    *   Реализована обратная связь для пользователя (сообщения об успехе/ошибке, отключение кнопки во время отправки).
    *   Конфигурация Firebase вынесена в отдельный файл `firebase-config.js` и добавлена в `.gitignore`.
*   **Конфигурация хостинга:** Создан `firebase.json` для деплоя сайта из папки `src`.
